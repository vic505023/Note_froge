# Улучшения UX редактора — Obsidian-стиль

## ✅ Что исправлено

### 1. **AI Assistant header** — изменён порядок элементов

**Было:**
```
[AI ASSISTANT] [→]
```

**Стало:**
```
[→] [AI ASSISTANT]
```

**Изменения в `Header.svelte`:**
```svelte
<!-- AI Panel section -->
<div class="header-section ai-section">
  {#if $uiStore.aiPanelOpen}
    <button class="icon-btn" ...>  <!-- Сначала иконка -->
      <svg>...</svg>
    </button>
    <h1 class="section-title">AI Assistant</h1>  <!-- Потом текст -->
  {:else}
    ...
  {/if}
</div>
```

**Результат:**
- Консистентность с sidebar (там тоже иконки слева)
- Лучшая визуальная группировка
- Иконка + текст как единый блок

---

### 2. **Центральный редактор** — убрано ощущение "карточки"

**Было:**
- Текст внутри отдельного контейнера (`.editor-content`)
- Ощущение рамки/блока
- Padding на внутреннем элементе
- max-width на контейнере

**Стало:**
- Текст лежит прямо на фоне редактора
- Как в Obsidian — единое полотно
- Без визуальных границ

**Изменения в `EditorPane.svelte`:**

Убран внутренний контейнер:
```svelte
<!-- Было -->
<div class="editor-wrapper">
  <div class="editor-content">  <!-- Лишний контейнер -->
    <Editor />
  </div>
</div>

<!-- Стало -->
<div class="editor-wrapper">
  <Editor />  <!-- Прямо в wrapper -->
</div>
```

**CSS:**
```css
.editor-wrapper {
  flex: 1;
  overflow: hidden;
  display: flex;
  justify-content: center;
  padding: 40px 32px;  /* Комфортный padding */
}
```

**Результат:**
- ✅ Нет ощущения "карточки"
- ✅ Текст прямо на фоне
- ✅ Единое полотно для письма

---

### 3. **Контент редактора** — центрирование и max-width

**Изменения в `Editor.svelte`:**

```css
.editor {
  width: 100%;
  max-width: 800px;      /* Ограничение ширины текста */
  height: 100%;
  margin: 0 auto;        /* Центрирование */
}
```

**Параметры:**
- max-width: 800px (оптимально для чтения)
- Центрирование через `margin: 0 auto`
- Padding через wrapper (40px vertical, 32px horizontal)

**Результат:**
- ✅ Текст не растянут на весь экран
- ✅ Комфортная ширина для чтения (~700-800px)
- ✅ Центрирован горизонтально

---

### 4. **Курсор (caret)** — акцентный цвет

**Было:**
- Цвет по умолчанию или неопределённый

**Стало:**
```css
:global(.cm-content) {
  caret-color: var(--accent) !important;  /* Зелёный акцент */
}

:global(.cm-cursor, .cm-cursor-primary) {
  border-left-color: var(--accent) !important;
  border-left-width: 2px !important;  /* Чуть толще */
}
```

**Цвет:** `--accent` (#4c8b6b) — приглушённый зелёный

**Результат:**
- ✅ Курсор заметен
- ✅ Не кислотный, мягкий акцент
- ✅ Хорошо виден на тёмном фоне

---

### 5. **CodeMirror фон** — прозрачный

**Было:**
```css
:global(.cm-editor) {
  background: var(--bg-primary) !important;
}
```

**Стало:**
```css
:global(.cm-editor) {
  background: transparent !important;  /* Прозрачный */
}
```

**Также:**
- Gutters: `background: transparent` (без бордера)
- Убран `border-right` у gutters

**Результат:**
- ✅ Редактор сливается с фоном
- ✅ Нет визуальных границ
- ✅ Единое полотно

---

### 6. **Отступы и spacing**

**`.cm-content`:**
```css
padding: 0;  /* Убран padding (было 16px 0) */
```

**`.cm-line`:**
```css
padding: 3px 0;  /* Минимальный (было 0 8px) */
```

**Результат:**
- Текст ближе к краям
- Меньше лишнего пространства
- Более натуральное ощущение письма
